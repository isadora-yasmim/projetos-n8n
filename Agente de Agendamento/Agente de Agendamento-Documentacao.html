01 · overview
Visão Geral do Fluxo
💡
O workflow recebe mensagens do Telegram, identifica se são texto, áudio ou outro tipo, processa com um AI Agent conectado ao banco de dados e retorna as respostas ao paciente.
📨
Telegram Trigger
Escuta mensagens recebidas no chat ID configurado. Dispara o workflow a cada nova mensagem.
trigger
🔀
Identifica o Tipo da Mensagem
Switch com 3 saídas: Texto (message.text existe), Áudio (message.voice existe), Outros (fallback para qualquer outro tipo).
switch
⚙️
Processamento por Branch
Cada tipo de mensagem segue um caminho diferente antes de convergir na padronização.
branches
📦
Padronização dos Dados
Nó Set único que normaliza as variáveis mensagem_usuario, session_id e origem independente do tipo de entrada.
set
🤖
AI Agent
Recebe a mensagem padronizada, consulta as tools do PostgreSQL e responde em JSON com as mensagens formatadas.
agent · gpt
✉️
Entrega no Telegram
As mensagens são separadas, enviadas com delay de 1s entre cada uma para simular digitação natural.
telegram · split · delay

03 · branches
Processamento por Tipo de Mensagem
📝 Texto
  • Identifica: message.text existe
  • Vai direto para Padronização dos dados
  • Usa message.text como mensagem
🎙️ Áudio
  • Identifica: message.voice existe
  • Get Arquivo de Áudio — busca file_path via API do Telegram
  • Download Áudio — baixa o arquivo binário
  • Transcribe a recording — transcreve em PT-BR via OpenAI
  • Vai para Padronização dos dados
🚫 Outros
  • Fallback para tipos não suportados
  • Send Mensagem Padrão — informa ao usuário que só aceita texto ou áudio
  • Fluxo encerrado — não chega ao agente

📋 System Prompt — Agente de Agendamento (Markdown)

🪪 Identidade

Você é o assistente virtual de agendamento de consultas do Sobrevidas. Sua única função é agendar consultas para pacientes cadastrados.

Se o paciente perguntar algo fora do escopo de agendamento, redirecione gentilmente para o tema.

🔧 Ferramentas disponíveis

ToolQuando usar
buscar_pacienteLogo após receber o nome completo
busca_agendamentoAntes de buscar horários — verificar se já existe agendamento
buscar_horariosApós confirmar que pode prosseguir com novo agendamento
salvar_agendamentoSomente após confirmação explícita do paciente

📋 Fluxo obrigatório

  1. Apresentação — Se apresente e peça o nome completo. Não execute nenhuma outra ação antes disso.
  2. Buscar paciente — Use buscar_paciente com o nome recebido. Sem ID válido, não prossiga.
  3. Verificar agendamento — Use busca_agendamento. Se existir, informe os dados e pergunte se quer criar um novo.
  4. Buscar horários — Use buscar_horarios. O campo range_horario define o intervalo (formato HH:MM-HH:MM).
  5. Sugerir horários — Apresente 3 opções distribuídas dentro do range, a partir da data atual. Use emojis 1️⃣ 2️⃣ 3️⃣.
  6. Confirmar — Antes de salvar, envie a confirmação e aguarde resposta explícita.
  7. Salvar — Use salvar_agendamento. O horario_fim = horario_inicio + 30 min.
  8. Encerrar — Envie os dados do agendamento e finalize cordialmente.

⚠️ Regras críticas

NUNCA confirme horário fora do range_horario retornado pela tool.
NUNCA invente dados de paciente, horário ou local — use apenas o retorno das tools.
NUNCA pule etapas do fluxo.

Sempre refira ao local usando "em" (nunca "na" ou "no").

📤 Formato de resposta

Responda sempre em JSON válido, sem texto fora do JSON. Divida em mensagens curtas.

{
  "mensagens": [
    { "mensagem": "texto aqui" }
  ]
}
06 · database tools
Tools PostgreSQL
🗄️
O agente possui acesso a 4 tools conectadas ao PostgreSQL. Todas usam $fromAI() para que o agente injete os parâmetros corretos em cada chamada.
Tool Descrição & SQL
buscar_paciente
 Busca o paciente pelo nome informado. Retorna id e nome_paciente. Sem resultado, o fluxo não avança.
SELECT id, nome_paciente FROM paciente_dados_sensiveis WHERE nome_paciente ILIKE '%' || $fromAI("nome_paciente") || '%' LIMIT 1;
busca_agendamento
 Verifica se o paciente já tem agendamento existente. Retorna o mais recente.
SELECT a.id, a.horario_inicio, a.horario_fim, e.nome_estabelecimento FROM agendamento_criado a JOIN estabelecimento e ON e.id = a.id_estabelecimento WHERE e.id_paciente = $fromAI("id_paciente") ORDER BY a.horario_inicio DESC LIMIT 1;
buscar_horarios
 Retorna o estabelecimento e o range_horario vinculado ao paciente. O range define o intervalo permitido para agendamentos.
SELECT e.id AS id_estabelecimento, e.nome_estabelecimento, e.range_horario FROM estabelecimento e WHERE e.id_paciente = $fromAI("id_paciente");
salvar_agendamento
 Insere o agendamento confirmado. O horario_inicio deve estar dentro do range_horario. O horario_fim é calculado como horario_inicio + 30 min.
INSERT INTO agendamento_criado (id_paciente, id_estabelecimento, horario_inicio, horario_fim, created_at) VALUES ( $fromAI("id_paciente"), $fromAI("id_estabelecimento"), '$fromAI("horario_inicio")', -- formato HH:MM:SS, dentro do range '$fromAI("horario_fim")', -- horario_inicio + 30 min NOW() ) RETURNING id, id_paciente, id_estabelecimento, horario_inicio, horario_fim;

08 · variables
Variáveis do Workflow
mensagem_usuario
string · Set: Padronização dos dados
Texto normalizado da mensagem — proveniente de transcrição (áudio) ou diretamente do Telegram (texto).
session_id
string · message.chat.id
ID único da conversa no Telegram. Chave da memória do agente.
origem
"audio" | "texto"
Indica a origem da mensagem. Determinado por presença de message.voice.
id_paciente
integer · via $fromAI()
ID do paciente retornado pela tool buscar_paciente. Usado em todas as queries subsequentes.
range_horario
string · "HH:MM-HH:MM"
Intervalo de funcionamento do estabelecimento. Define os limites para sugestão e validação de horários.
id_estabelecimento
integer · via buscar_horarios
ID do estabelecimento vinculado ao paciente. Necessário para salvar o agendamento.
